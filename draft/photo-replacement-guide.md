## 差し替え手順

### 1. 新しい写真を用意する

差し替えたい写真をレタッチ・書き出しした状態で用意します。

### 2. 画像を書き出す（解像度・圧縮の設定）

ファイル容量が大きくなりすぎないよう、以下の基準で書き出してください。

| 項目 | 設定値 |
|---|---|
| 形式 | JPEG（ロゴのみ PNG） |
| 色空間 | sRGB |
| 解像度（pixel/inch） | 72 dpi |
| 画像の幅 | 上の一覧表の「サイズ」に合わせる |
| JPEG 品質（圧縮率） | 70〜80%（Photoshop の場合は 8〜10 程度） |
| 容量の目安 | 1枚あたり 200KB〜400KB |

**ポイント**

- 元写真の解像度が高い場合（例：5000px 以上）は、必ず上の表のサイズにリサイズしてから書き出してください。解像度が大きいまま書き出すと容量が数MBになり、ページの読み込みが遅くなります。
- 72 dpi で十分です。300 dpi などの印刷用設定のままにしないでください。
- JPEG 品質は 80% を超えると容量が急増しますが、見た目の差はほとんどありません。70〜80% が画質と容量のバランスが良い範囲です。
- Lightroom から書き出す場合は「画質 75」「解像度 72」「長辺 1050px に合わせる」で設定すると上記の基準を満たせます。

### 3. ファイル名を合わせる

差し替えたい場所に対応するファイル名にリネームします。

- 例：ヒーロー画像の3枚目を差し替えたい → `hero_3.jpg`
- ファイル名は**完全一致**させてください（大文字・小文字も区別されます）

### 4. FTPでアップロードする

1. FTPソフトでサーバーに接続する
2. サーバー上の `/assets/images/` フォルダを開く
3. 該当ファイルを上書きアップロードする

### 5. サイトを確認する

- サイト（ http://little-seeds.ayako-suzuki.net/ ）を開く
- 画像が変わらない場合は、ブラウザのキャッシュをクリアして再読み込みする
  - Windows： Ctrl + Shift + R
  - Mac： Cmd + Shift + R
  - スマートフォン：ブラウザの設定からキャッシュを削除

## 注意事項

- **アスペクト比を揃える**：画像は CSS で自動トリミングされるため、表示が大きく崩れることはありませんが、既存の画像と同じ縦横比を保つと意図しない切り取りを防げます。
- **バックアップ**：上書き前に、サーバー上の古い画像をローカルにダウンロードして保管しておくと安心です。
- **OGP 画像**：SNS でシェアされたときのサムネイルには `hero_1.jpg` が使用されます。この画像を差し替えた場合、SNS 側のキャッシュが更新されるまで（数日〜数週間）古い画像が表示されることがあります。
