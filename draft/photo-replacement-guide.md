# サイト写真差し替え手順ガイド

## はじめに

このサイトは静的HTMLで構成されており、CMSはありません。写真の差し替えはファイルを直接置き換える方法で行います。`main` ブランチに変更をプッシュすると、GitHub Actions により自動的にデプロイされます。

---

## 差し替え手順

### 方法A：同じファイル名で差し替える場合（推奨）

HTMLの編集が不要なため、もっとも簡単な方法です。

1. **新しい写真を用意する**
   - 上の表の「推奨サイズ」を参考に、同じアスペクト比の画像を用意してください
   - 形式はJPEGにしてください（拡張子 `.jpg`）
   - ファイルサイズの目安：1枚あたり 200KB〜800KB 程度

2. **ファイル名を合わせる**
   - 差し替えたい場所に対応するファイル名にリネームします
   - 例：ヒーロー画像の3枚目を差し替える場合 → `hero_3.jpg`

3. **ファイルを置き換える**
   - `web/assets/images/` フォルダ内の該当ファイルを新しいファイルで上書きします

4. **変更をプッシュする**
   ```bash
   git add web/assets/images/hero_3.jpg
   git commit -m "ヒーロー画像3枚目を差し替え"
   git push origin main
   ```

5. **デプロイを確認する**
   - GitHub の Actions タブで、デプロイが正常に完了したことを確認します
   - サイト（ http://little-seeds.ayako-suzuki.net/ ）を開いてブラウザのキャッシュをクリア（Ctrl+Shift+R）し、画像が反映されていることを確認します

---

### 方法B：ファイル名を変えて差し替える場合

ファイル名を変更したい場合は、HTMLも一緒に修正する必要があります。

1. **新しい画像ファイルを `web/assets/images/` に追加する**

2. **`web/index.html` を編集する**
   - テキストエディタで `web/index.html` を開き、該当する `<img>` タグの `src` 属性を新しいファイル名に書き換えます
   - 例：
     ```html
     <!-- 変更前 -->
     <img src="assets/images/concept.jpg" alt="眠っている赤ちゃん" class="section__image">

     <!-- 変更後 -->
     <img src="assets/images/concept-new.jpg" alt="眠っている赤ちゃん" class="section__image">
     ```
   - 必要に応じて `alt` 属性（画像の説明文）も更新してください

3. **古い画像ファイルを削除する**（任意）
   - 不要になった古いファイルは削除して構いません

4. **変更をプッシュする**
   ```bash
   git add web/assets/images/ web/index.html
   git commit -m "コンセプト画像を差し替え"
   git push origin main
   ```

## スライドショーの枚数を変更したい場合

ヒーロー画像（トップのスライドショー）の枚数を増減する場合は、以下の追加作業が必要です。

### 枚数を減らす場合

`web/index.html` のヒーローセクションから不要な `<img>` タグを削除します。

```html
<!-- この行を削除すると5枚目が消える -->
<img src="assets/images/hero_5.jpg" alt="ニューボーンフォト" class="hero__slide">
```

### 枚数を増やす場合

`web/index.html` のヒーローセクションに `<img>` タグを追加し、対応する画像ファイルを配置します。

```html
<img src="assets/images/hero_6.jpg" alt="ニューボーンフォト" class="hero__slide">
```

スライドショーの JavaScript（`web/js/main.js`）は自動で枚数を検出するため、JSの修正は不要です。

## 注意事項

- **画像のアスペクト比を揃える**：CSSで `aspect-ratio` と `object-fit: cover` が設定されているため、異なるアスペクト比の画像でも表示は崩れませんが、意図しないトリミングが発生することがあります。なるべく既存画像と同じ比率を維持してください。
- **ファイルサイズに注意する**：画像が大きすぎるとページの読み込みが遅くなります。1枚あたり 800KB 以下を目安にしてください。
- **OGP画像**：SNSでシェアしたときに表示されるサムネイルは `hero_1.jpg` が使われています。この画像を差し替えた場合、SNS側のキャッシュが更新されるまで古い画像が表示されることがあります。
- **ブラウザキャッシュ**：差し替え後に画像が変わらない場合は、ブラウザのキャッシュをクリアしてください（Ctrl+Shift+R または Cmd+Shift+R）。
- **バックアップ**：差し替え前に、念のため古い画像ファイルを別の場所に保存しておくことをおすすめします。
